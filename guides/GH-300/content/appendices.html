C. Data flow & retention at a glance

StageWhat happens
1. Context captureEditor gathers open files, the current selection, neighboring tabs, and comments.
2. Prompt buildingContext is assembled and trimmed to fit the model's context window.
3. Proxy (pre-filter)Request passes through the Copilot proxy; safety/toxicity/PII filtering is applied.
4. Model inferenceThe LLM generates candidate completions.
5. Post-processingOutput filtering, optional public-code matching, then the suggestion is returned.
RetentionEditor completions aren't retained after serving; Business/Enterprise prompts & suggestions are not used to train the models.
Gotcha: The proxy filters before and after the model. "No training on your data" is a Business/Enterprise guarantee; individual plans offer an opt-out toggle.

D. Prompt-crafting cheat sheet

Make prompts specific, contextual, and iterative

  • State the goal clearly — what you want, in what language, with which constraints.
  • Give relevant context — keep the right files open; select the code you're referring to; use good variable/function names and comments.
  • Break big asks into steps — smaller prompts produce better, reviewable output.
  • Use examples (few-shot) when you need a specific output shape or pattern; zero-shot is fine for simple tasks.
  • Iterate — refine follow-up prompts; start a new chat to drop stale context.
  • Provide sample data / expected output to steer format.
Remember: Copilot uses your prompt plus editor context plus chat history. Irrelevant open files or a long stale conversation can degrade suggestions.

Content exclusionsSuggestions matching public code
PurposeKeep specified files/repos out of Copilot's contextBlock suggestions that match public code
EffectNo suggestions shown for excluded content; it isn't used as contextSuggestions matching ~150 characters of public code are blocked

Gotcha: Content exclusions limit context; the public-code filter limits output. They solve different problems — an exam item may test whether you pick the right one.

F. Responsible AI & validation checklist

Six responsible-AI principles

  • Fairness — treat all people equitably.
  • Reliability & safety — perform safely and consistently.
  • Privacy & security — protect data and respect consent.
  • Inclusiveness — empower and engage everyone.
  • Transparency — be understandable and explainable.
  • Accountability — people remain responsible for AI systems.

Before you accept a suggestion

  • Read and understand it — never accept code you can't explain.
  • Test and verify — run it; check edge cases and security.
  • Check licensing/IP — enable the public-code filter where required.
  • Own the result — you are accountable for accepted code, not Copilot.
