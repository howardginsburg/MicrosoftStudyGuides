About the exam

Exam facts

  • Passing score: 700 (on a scale to 1000).
  • Audience: developers, engineers, and technical leads who use GitHub Copilot day to day and want to prove they can apply it effectively, responsibly, and securely.
  • What you should know: how Copilot works across surfaces (IDE, Chat, CLI, GitHub.com), prompt engineering, Copilot's data flow and architecture, responsible-AI practices, and organization-level configuration (content exclusions, policies, public-code filtering).
  • Question styles: multiple-choice and multi-select scenario questions. Most items target generally available features; you will often need to choose the right feature, mode, or setting for a scenario rather than just recall a fact.
  • Plan awareness matters: be able to tell which capabilities belong to Copilot Free, Pro/Pro+, Business, and Enterprise.

Skills measured — blueprint (weighting)

DomainWeightWhat it covers
1. Use GitHub Copilot responsibly15–20%Responsible-AI principles; risks & limitations of generative AI; potential harms and mitigations; validating AI output; operating Copilot responsibly.
2. Use GitHub Copilot features25–30%Copilot in the IDE (inline, chat, CLI, agent mode); the Copilot CLI; features & capabilities (Agent Mode, Copilot Edits, MCP, code review, Spaces, PR summaries, instructions/prompt files); org-wide settings, policies, audit logs, and the REST API.
3. Understand GitHub Copilot data and architecture10–15%Data usage, flow, and sharing; input processing and prompt building; proxy filtering and post-processing; the code-suggestion lifecycle; limitations of LLMs and Copilot.
4. Apply prompt engineering and context crafting10–15%Prompt structure and context; how context is determined; zero-shot and few-shot prompting; best practices; prompt engineering principles; prompt process flow and chat history.
5. Improve developer productivity with GitHub Copilot10–15%Code generation, refactoring, and documentation; accelerating learning and reducing context switching; sample data and modernizing legacy code; unit & integration tests; edge cases and assertions; security and performance suggestions.
6. Configure privacy, content exclusions, and safeguards10–15%Content exclusions and editor settings; ownership and limitations of outputs; public-code matching filter; troubleshooting suggestions and exclusions.

Domain 2 is the largest slice of the exam — invest the most preparation time there.
