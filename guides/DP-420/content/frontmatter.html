About the exam

Exam facts

  • Passing score: 700 (on a scale to 1000).
  • Role: software / cloud developer — you design, implement, and monitor cloud-native applications that store and manage data in Azure Cosmos DB, and integrate the solution with other Azure services.
  • Primary API: the exam centers on Azure Cosmos DB for NoSQL. You should be fluent writing its SQL query language, reading C# or Java SDK code, authoring server-side JavaScript, interpreting JSON, and using PowerShell / Azure CLI and ARM/Bicep.
  • Question styles: multiple-choice, multi-select, drag-and-drop/ordering, and scenario/case studies. Most questions target GA features; commonly used preview features may appear.
  • Expect to reason about tradeoffs — partition keys, throughput modes, consistency levels, indexing policies, and backup strategies — as often as to recall a fact.

Skills measured — blueprint (weighting)

DomainWeightWhat it covers

Design and implement data models35–40%Non-relational modeling (embed vs reference, denormalization, TTL, versioning); partitioning strategy (partition keys, synthetic & hierarchical keys, distribution); sizing & scaling (serverless/provisioned/autoscale/free tier); SDK connectivity; the NoSQL SQL language; SDK data access (point ops, patch, batch, bulk, ETags, pagination, 429s); server-side JavaScript.
2. Design and implement data distribution5–10%Replication strategy (when to distribute, failover, the five consistency levels and their tradeoffs); multi-region writes and conflict resolution.
3. Integrate an Azure Cosmos DB solution5–10%Analytical workloads (Fabric mirroring, analytical store, Synapse Link, Spark connector, analytical-store CDC, Fabric Warehouse time travel); cross-service solutions with Change Feed, Azure Functions, Event Hubs, and Azure AI Search.
4. Optimize an Azure Cosmos DB solution15–20%Query performance and RU cost; the integrated cache; change feed design (processor, estimator, design patterns); indexing strategy (index types, composite indexes, read- vs write-heavy tuning).
5. Maintain an Azure Cosmos DB solution25–30%Monitoring & troubleshooting (Azure Monitor metrics, resource logs, partition analysis); backup & restore (periodic vs continuous / PITR); security (network, RBAC control vs data plane, encryption, CMK, Always Encrypted); data movement; DevOps (declarative ARM/Bicep, PowerShell/CLI).

Domain 1 is the largest slice of the exam and Domain 5 is second — invest the most preparation time in modeling, partitioning, and the maintain/operate skills.
